## Objective
Build a rigorous, comprehensive test suite for all features and their associated functions.

## Requirements
- Write only honest, meaningful tests that verify actual functionality (no mock tests or placeholder assertions)
- Include unit tests for all functions and edge cases
- Include Playwright end-to-end tests for all user-facing features and interactions
- Test every user-accessible feature and workflow
- Add comprehensive, descriptive logging for all errors and state transitions
- Do not use shortcuts, hacks, or tests that artificially inflate pass rates

## Success Criteria
If all tests pass with 100% coverage, the entire application is production-ready and all features work correctly from both technical and user perspectives.do not include tests that pass if the function fails. test passage only ever means full functionality.
