## Objective
Build a rigorous, comprehensive test suite for the specified feature and all associated functions.

## Requirements
- Write only honest, meaningful tests that verify actual functionality (no mock tests or placeholder assertions)
- Include unit tests for all functions and edge cases
- Include Playwright end-to-end tests simulating real user interactions
- Add comprehensive, descriptive logging for all errors and state transitions
- Ensure tests are thorough enough that 100% pass rate guarantees the feature works correctly
- Do not use shortcuts, hacks, or tests that artificially inflate pass rates

## Success Criteria
If all tests pass, the feature is production-ready and functions as designed from both technical and user perspectives.
